# Игра "Угадай число"
В данном решении демонстрируются принципы SOLID: 
1. Single Responsibility Principle (Принцип единственной ответственности) - каждый класс выполняет только одну задачу.
	Каждый из классов NumberGuessingGame, RandomNumberGenerator, BasicHintStrategy и AdvancedHintStrategy выполняет только одну задачу
2. Open/Closed Principle (Принцип открытости/закрытости) - классы должны быть открыты для расширения, но закрыты для модификации.
	Интерфейс IRandomNumberGenerator позволяет изменять способ генерации случайных чисел без изменения логики игры. Но для более показательной демонстрации
	реализована система подсказок. Чтобы реализовать продвинутую систему подсказок, мы не модифицировали имеющуюся а расширили, используя еще одну реализацию IHintStrategy.
3. Liskov Substitution Principle (Принцип подстановки Барбары Лисков) - объекты в программе можно заменять их наследниками без изменения свойств программы.
	Конструктор класса NumberGuessingGame принимает в себя интерфейсы и может принимать любую реализацию производных классов без поломки функционала.
4. Interface Segregation Principle (Принцип разделения интерфейса) - много специализированных интерфейсов лучше, чем один универсальный.
	Нет универсальных интерфейсов с общим набором методов. Созданы специализированные интефейсы IHintStrategy, INumberGuessingGame и IRandomNumberGenerator
5. Dependency Inversion Principle (Принцип инверсии зависимостей) - модули высокого уровня не должны зависеть от модулей низкого уровня.
	Класс NumberGuessingGame зависит от абстракции IRandomNumberGenerator и IHintStrategy, а не от конкретной их реализации. 
    Так же как реализация RandomNumberGenerator, BasicHintStrategy и AdvancedHintStrategy зависит от этих же интерфейсов.

Настройки расположены в файле appsettings.json
